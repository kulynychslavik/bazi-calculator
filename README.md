# BaZi — перевірочний сайт (4 стовпи + денний майстер)

Простий локальний сайт, щоб **звірити розрахунок БаЦзи з bazi-calculator.com**
перед тим, як переносити логіку в Telegram-бота (SendPulse / API).

## Що робить

Ввід: дата народження, час народження, місто/країна.
Вивід: 4 стовпи БаЦзи (рік / місяць / день / година) + денний майстер — **у двох варіантах**:

1. **Простий** — час як на годиннику, без поправки довготи (крок 1 за ТЗ).
2. **LMT / довгота** — справжній місцевий сонячний час: геокодинг міста → координати →
   історичний часовий пояс і DST на дату → поправка довготи.

Так одразу видно, який варіант збігається з bazi-calculator.com — без переробки коду.

## Запуск

```bash
pip install -r requirements.txt
python3 app.py
```

Відкрити http://127.0.0.1:5001

## Як перевіряти (за ТЗ)

1. Проженіть 20 карт через «Простий варіант» і звірте з bazi-calculator.com.
2. Якщо збіг 100% — поправку довготи не вмикаєте, у бот переносите простий розрахунок.
3. Якщо ні — звіряйте з варіантом «LMT / довгота».

## Бібліотеки

- `lunar_python` — розрахунок БаЦзи: `Solar.fromYmdHms(...).getLunar().getEightChar()`.
  Денний майстер = `getDayGan()` (небесний стовбур дня).
- `geopy` (Nominatim) — місто → координати. Потрібен інтернет.
- `timezonefinder` + `pytz` — історичний часовий пояс і DST на дату народження.

## Перенесення в бота

Логіка з `app.py` (функції `geocode_city`, `resolve_timezone`, `eight_char`,
`lmt_correction_minutes`) переноситься в окремий ендпоінт `POST /bazi`, який
викликає SendPulse через блок «Запрос API». SendPulse сам Python не виконує —
він лише шле дані на цей backend і показує відповідь.
